Els tres passos de la investigació

01
Fase inicial

Generació del Misteri

En el moment que inicies una partida, el sistema defineix en segon pla la Solució Secreta: una combinació aleatòria d'un personatge, una arma i una habitació. Aquesta informació queda guardada al servidor Django i tu, com a investigador, no tens cap accés directe.

⚙ Backend (Django ORM) genera i desa la Solució Secreta de forma anònima a la base de dades.
02
Fase central

L'Interrogatori

A la interfície (Vue.js) veuràs tres llistes: Personatges, Armes i Habitacions. Selecciona un element de cada categoria, formula la teva teoria i prem el botó ACUSAR. La petició s'envia via fetch a l'API de Django sense recarregar la pàgina.

⚙ POST /api/acusar/ → El frontend envia { personatge, arma, habitacio } i rep la resposta en temps real.
03
Fase final

El Veredicte

El sistema compara la teva acusació amb la Solució Secreta i retorna immediatament el resultat. Si has encertat els tres elements, el cas es tanca. Si no, la investigació continua: analitza les pistes que el sistema et proporciona i torna a intentar-ho.

⚙ Resposta JSON: { "correcte": true/false, "pistes": "Arma correcta, lloc no" }

Restriccions tècniques

⚖️

L'ORM és la Llei

Tota interacció amb la base de dades s'ha de fer obligatòriament a través del Django ORM. No s'accepta SQL cru sota cap circumstància.

🚫

Prohibit Recarregar

La vista de Vue.js és completament reactiva. Totes les accions —obtenir llistes, acusar— s'han de realitzar via JavaScript fetch, sense cap recàrrega de pàgina.

🏛️

Silos Tecnològics

Vue.js s'encarrega exclusivament de la interfície i la interactivitat. Django gestiona exclusivament la seguretat de les dades i l'API. Cap dels dos envaeix el territori de l'altre.

Resultats possibles

✔

Correcte → El Cas es Tanca

Has identificat correctament el personatge, l'arma i l'habitació. El sistema confirma l'acusació i declara el cas resolt. La teva reputació com a detectiu queda intacta.

✘

Incorrecte → La Investigació Continua

Un o més elements de la teva acusació no coincideixen amb la Solució Secreta. El sistema et pot proporcionar pistes parcials si has assolit el nivell Inspector o superior.

Notes del Comissari

  • Comença pel mínim funcional (MVP). No busquis l'Excel·lent abans de tenir un joc que funcioni per a l'Aprovat.
  • L'API és el teu pont. Prova els endpoints (ex: /api/personatges/) directament al navegador abans de programar el Vue.
  • Revisa les teves eines. Les URLs, la consola del navegador (F12) per errors de Vue, i el terminal de Django. Allà hi ha les pistes.
  • Totes les dades base s'han de crear prèviament des de la interfície de Django Admin: personatges, armes, habitacions i la solució.
  • Pensa en capes. Primer els models, després l'API, finalment el frontend. No saltis passos o et perdràs.
  • Prova cada endpoint per separat. Confirma que el GET funciona abans d'implementar el POST d'acusació.
